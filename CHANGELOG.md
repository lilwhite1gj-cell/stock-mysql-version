# 更新日志 (Changelog)

所有重要变更均会记录在此文件中。

> ## 📌 维护约定（必读）
> - **版本号 `1.1.N` 由 CI 自动生成**：每次推送 `main`，GitHub Actions 会根据总提交数自动改写 `package.json` 的 `version`、前端 `appVersion` 变量与版本徽章，无需手动修改。
> - **更新日志内容需手动维护**：CI 仅会把已存在的 `## [1.1.N] - 日期` 标题行的日期补成当天，**不会自动总结"改了什么"**。
> - **开发流程**：每次新增/修改功能后，请在本文件顶部手动添加一节 `## [1.1.N] - YYYY-MM-DD` 并填写变更要点（参考下方历史格式），再提交推送。版本号 N 以部署后 `/api/version` 实际显示为准。

## [1.1.32] - 2026-07-23

### 修复
- 🐛 修复注册API返回500错误：saveLocalData()在data目录不存在时崩溃
- 🐛 修复登录返回401问题：添加诊断端点排查MySQL连接状态
- 🛡️ saveLocalData()自动创建data目录（递归创建）
- 🛡️ 注册API本地JSON路径添加try-catch错误处理

### 新增
- 🔍 新增 /api/health 诊断端点，显示数据库模式、MySQL连接状态、用户数量等

## [1.1.0] - 2026-07-22

### 新增
- 🏷️ 版本号与更新日志系统，支持前端页面查看版本信息
- 🔄 GitHub Actions 自动部署工作流（推送到 main 分支自动同步至 Hostinger FTP）
- 🔒 服务端安全加固：CORS 收敛、请求限流、安全响应头
- 🛡️ 前端 XSS 防护：输入转义、上传文件类型校验

### 修复
- 🐛 修复删除流水记录时"撤销失败"无具体错误原因的问题，现在会显示服务端返回的详细错误信息
- 🐛 修复负库存校验删除入库记录时前端只显示通用提示的问题

### 变更
- ⬆️ cloudinary 依赖曾升级至 v2.7.0（修复 CVE-2025-12613 任意参数注入），后因与 multer-storage-cloudinary 依赖冲突降级回 1.x（当前固定 1.41.3）
- ⬆️ express 升级至 v5.2.1
- ⬆️ multer 升级至 v2.2.0

## [1.0.0] - 2026-07-10

### 核心功能
- 📦 库存台账管理：产品 CRUD、分类、SKU 货号、规格材质
- 📊 仪表盘看板：库存总览、收发统计、趋势图表
- 🔄 收发明细：入库/出库流水记录、图片凭证、撤销与库存回滚
- 🏭 工厂档案：供应商信息管理
- 👥 客户档案：客户信息与联系方式管理
- 🔐 用户认证：注册/登录、JWT Token、角色权限（admin/staff）
- 👑 账号管理：管理员可增删用户、调整角色权限
- 🏷️ 品类维护：自定义产品分类
- ⚠️ 库存预警：低库存自动提醒（阈值 10）
- 💱 汇率转换：CNY/USD 双币种支持，实时汇率查询
- 📤 数据导出：库存 CSV 导出、流水 CSV 导出
- 💾 数据备份：JSON 全量备份导出/导入恢复
- 🖼️ 图片上传：Cloudinary 云存储集成
- 📱 响应式设计：桌面端侧边栏 + 移动端底部导航
- 🗄️ MySQL 数据库：生产环境 MySQL + 开发环境 JSON 文件双模式
